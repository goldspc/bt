# Космический бой 10×10×10

Многопользовательская версия игры с веб-интерфейсом (мастер игры и игроки
подключаются через браузер) и набором вспомогательных скриптов.

```
.
├── game/     # всё, что нужно, чтобы поднять игровой сервер и играть
│             # (см. game/README.md для запуска, пароля GM, тестов)
├── tools/    # вспомогательное: LAN-клиенты, симулятор, турнир, UI-демо
└── README.md
```

- **Как поднять и играть** — [`game/README.md`](game/README.md).
- **Симулятор / турнир / LAN-GUI** — [`tools/README.md`](tools/README.md).

## Как подключить друзей к игре

После запуска `game/run_lan.sh` (или `run_lan.ps1`) сервер слушает порт
`8000` на всех интерфейсах. Любой, кто сетевым путём может достучаться до
этого порта, откроет игру в браузере:

```
http://<IP-хоста>:8000/
```

Мастер игры заходит в `/admin`, вводит пароль (`GM_PASSWORD`), создаёт игру
— сервер выдаёт `ID` (6 символов) + `ключ` (4 цифры). Пересылаешь эти две
строки или ссылку `http://<IP>:8000/play?c=ID&k=KEY` друзьям.

Ниже — три способа сделать этот `<IP-хоста>` достижимым для твоего друга.

### Способ 1. Та же локальная сеть (Wi-Fi/LAN)

Самый простой случай: все подключены к одному роутеру (домашний Wi-Fi или
один свитч).

1. Запусти игру на хосте:

   ```bash
   cd game
   ./run_lan.sh     # Linux/macOS
   # .\run_lan.ps1  # Windows PowerShell
   ```

2. Узнай IP хоста в локалке:

   - Windows: `ipconfig` → строка `IPv4-адрес` у активного адаптера.
   - Linux: `hostname -I`.
   - macOS: `ipconfig getifaddr en0` (Wi-Fi) / `en1` (Ethernet).

3. Разреши входящие на `8000/tcp` в фаерволе хоста:

   - Windows: Панель управления → Брандмауэр Защитника Windows → «Разрешить
     работу приложения» → добавить `python.exe` (или правило для порта
     8000 TCP, профили «Частная» и «Доменная»).
   - Linux (ufw): `sudo ufw allow 8000/tcp`.

4. Друг открывает в браузере `http://<IP>:8000/` и заходит в `/play` по
   выданной тобой ссылке.

Если вы в разных сетях (например, разные квартиры) — см. способы 2 и 3.

### Способ 2. Radmin VPN (виртуальная общая сеть поверх интернета)

Radmin VPN бесплатный и создаёт виртуальную LAN — все участники получают
IP-адреса `26.x.x.x`, и вы друг для друга как будто сидите в одной
локалке.

1. Скачай и поставь [Radmin VPN](https://www.radmin-vpn.com/) на хосте и
   на машине друга.
2. На хосте: **Сеть → Создать сеть** → задай имя и пароль → нажми «Создать».
3. Сообщи другу имя сети + пароль. Он: **Сеть → Присоединиться к сети**.
4. После подключения Radmin выдаёт каждой машине адрес `26.x.x.x`. На
   хосте посмотри свой (в окне Radmin VPN возле зелёного шарика).
5. Запусти игру как обычно:

   ```bash
   cd game
   ./run_lan.sh
   ```

6. Другу дай ссылку с Radmin-адресом хоста:

   ```
   http://26.12.34.56:8000/
   ```

   и ссылку на его игрока `http://26.12.34.56:8000/play?c=XXXX&k=YYYY`.

Важно: **открой порт 8000 в фаерволе хоста** (см. шаг 3 из способа 1),
иначе Radmin-трафик не пройдёт.

### Способ 3. Публикация в интернет без VPN (ngrok / Cloudflare Tunnel)

Когда у тебя нет белого IP и не хочется настраивать VPN, подойдёт
тоннель. Для друга это просто ссылка вида `https://xxxx.ngrok-free.app`.

#### 3а. ngrok

1. Зарегистрируйся на [ngrok.com](https://ngrok.com/), поставь клиент
   (`choco install ngrok`, `brew install ngrok/ngrok/ngrok` или скачай с
   сайта).
2. Залогинься: `ngrok config add-authtoken <твой-токен>`.
3. Подними игру локально:

   ```bash
   cd game
   ./run_lan.sh
   ```

4. В отдельном терминале:

   ```bash
   ngrok http 8000
   ```

   ngrok выдаст строку `Forwarding https://xxxx.ngrok-free.app -> http://localhost:8000`.

5. Отправь другу `https://xxxx.ngrok-free.app/play?c=XXXX&k=YYYY`.

#### 3б. Cloudflare Tunnel (бесплатно, без регистрации для быстрого теста)

```bash
# Один раз поставить:
# Windows: winget install --id Cloudflare.cloudflared
# macOS:   brew install cloudflared
# Linux:   sudo apt install cloudflared

cloudflared tunnel --url http://localhost:8000
```

Клиент сам выведет что-то вроде
`https://random-words.trycloudflare.com` — это и есть публичная ссылка.

> **Совет по безопасности.** Любой публичный тоннель открывает игру всему
> интернету. Обязательно задай нетривиальный `GM_PASSWORD` и не выдавай
> всем подряд код/ключ игры — иначе чужие игроки могут набиваться в твою
> команду.

## Требования

- Python **≥ 3.10**.
- Для `tools/*.py` на Linux дополнительно: `python3-tk`.
- Порт **8000/tcp** должен быть свободен и открыт в фаерволе хоста.
